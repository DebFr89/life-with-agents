# Sky Career ✈️

A career-mode airplane flying game for iPhone/iPad, built as a **self-contained HTML5
web game** (vanilla JS, Canvas 2D, no build step, no dependencies). Installs to the
home screen as a PWA — fullscreen, landscape, offline.

Start as a student pilot in a Cessna 172, fly route contracts for cash, and progress
through real pilot licenses (Student → PPL → CPL → Instrument/Multi → ATPL), each
unlocking faster, higher-capacity aircraft and tougher, more lucrative routes.

## Play it

**On a computer (development / quick try):**

```bash
cd ~/sky-career
python3 -m http.server 8000
# open http://localhost:8000
```

Controls on desktop: **arrow keys** steer & pitch, **hold Space** for throttle.

**On your iPhone/iPad:**

Tilt steering needs a **secure context (HTTPS)** on iOS, so deploy the folder to any free
static host and open the HTTPS URL on the device:

- **GitHub Pages** — push this folder to a repo, enable Pages on the branch root.
- **Netlify / Cloudflare Pages** — drag-and-drop the folder (no build command).

Then on the device: tap **Enable Tilt Controls** → grant motion access → fly by tilting,
**hold the screen** for throttle. Tap the **Share → Add to Home Screen** to install it as a
fullscreen app. (Over plain `http://<lan-ip>` tilt won't be granted, but the on-screen
touch stick still works.)

## How a flight works

1. Pick a contract in the **Hub**, review it in the **Briefing**, start the flight.
2. **Take off** (hold throttle to build speed, pitch up to climb), **cruise** through the
   scoring gates while managing fuel and dodging weather/traffic, then **line up and
   descend** on **approach** and touch down gently.
3. Landing is graded (Perfect / Good / Hard / Crash) — better landings + more gates pay
   more. Spend earnings in the **Hangar**; earn flight hours toward the next license, then
   pass its **check-ride** in the **License** center to advance.

## Controls

| Mode | Steer / pitch | Throttle |
|------|---------------|----------|
| Tilt (iOS) | tilt the device | hold anywhere on screen |
| Touch | left-side virtual stick | hold the right side |
| Keyboard | arrow keys / WASD | hold Space |

## Project layout

```
index.html  manifest.webmanifest  sw.js  icons/  tools/gen_icons.py
src/
  core/    Game loop, canvas/DPR, scene manager, audio (WebAudio, synthesized)
  input/   tilt + touch + keyboard, merged by InputManager
  render/  Pseudo3D projection, background, sprites, entities, particles
  world/   flight model, world/route builder, collisions, landing judge
  scenes/  Boot · Menu · Hub · Hangar · License · Briefing · Flight · Results · Settings
  career/  save store (localStorage), economy, progression
  ui/      theme, canvas widgets, in-flight HUD
  data/    config + aircraft / licenses / routes tables  ← all balancing lives here
```

All graphics are drawn procedurally and all audio is synthesized — there are **no binary
asset files** to load (icons are generated by `tools/gen_icons.py`).

## Tuning & content

Everything balance-related is data-driven in `src/data/`. Add an aircraft, license, or
route by editing the tables there; add visual/physics feel via `src/data/config.js`.

> Note: after changing any file, bump `CACHE_VERSION` in `sw.js` so the PWA serves the
> update (the service worker is cache-first). During local dev you can also just clear
> site data / unregister the SW.
